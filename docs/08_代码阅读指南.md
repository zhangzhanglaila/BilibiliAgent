## 预备知识：核心概念入门

### 什么是 LLM？

LLM（Large Language Model，大语言模型）是一个神经网络，训练于海量文本，能够根据输入的文本（称为"提示词"）预测输出文本。你可以把它理解为一个**极其强大的文本补全器**——你给它一段文字，它预测接下来最可能写什么。

调用示例：
```python
llm = LLMClient()  # 配置好 API Key 和模型后
response = llm.invoke("你好，请介绍一下B站")
# response = "B站是中国知名的视频弹幕网站..."
```

本项目的 LLM 底层由 `llm_client.py` 中的 `LLMClient` 类封装。

### 什么是 LangChain？

LangChain 是一个框架，它让开发者可以方便地将 LLM 与外部工具、数据结合起来。LangChain 本身也是一个 Python 库，本项目的 `llm_client.py` 底层用了 LangChain 的 `ChatOpenAI` 等组件。

**你不需要深入学习 LangChain 也能理解本项目**，因为大部分核心逻辑是用更直接的方式封装在 `agents/llm_workspace_agent.py` 中的。只需知道：LangChain 负责"让 LLM 听懂指令并调用工具"这件事。

### 什么是 ReAct Agent？

这是本项目最核心的编程范式，必须理解。

ReAct = **Reasoning（推理）+ Acting（行动）**

简单说：Agent 循环执行以下步骤——
1. **思考**（Reasoning）：LLM 看到当前状态，决定"我应该做什么"
2. **行动**（Acting）：LLM 选择并调用一个工具（如搜索、检索）
3. **观察**（Observation）：工具返回结果，LLM 看到结果后继续思考
4. **循环**：直到 LLM 认为任务完成，返回最终结果

这个循环由 `LLMWorkspaceAgent.run_structured()` 实现。

### 什么是工具（Tool）？

工具是 Agent 可以调用的外部能力。在本项目中，工具包括：

| 工具名 | 作用 |
|--------|------|
| `retrieval` | 从本地知识库（Chroma）检索 B站爆款案例 |
| `web_search` | 联网搜索最新内容 |
| `video_briefing` | 解析 B站视频链接，获取视频信息 |
| `hot_board_snapshot` | 获取 B站热门榜 |
| `creator_briefing` | 生成创作者简报 |

工具在 `web/services/llm.py` 的 `build_workspace_agent_tools()` 中定义。

### 什么是 Prompt？

Prompt（提示词）是给 LLM 的输入文本，它告诉 LLM：
- 当前是什么任务
- 有哪些规则需要遵守
- 有哪些工具可以使用
- 期望什么样的输出格式

Prompt 分为两类：
- **System Prompt（系统提示词）**：设定 Agent 的身份和全局规则
- **User Prompt（用户提示词）**：具体的任务输入

**阅读指引**：这个函数是视频分析的"总指挥"。它的职责是：

- analysis: 对象，必须包含 analysis_points, benchmark_analysis...

**⑤ ReAct 执行循环**（核心！）：
```python
result = agent.run_structured(
    task_name="module_analyze",
    task_goal=VIDEO_ANALYZE_TASK_GOAL,
    user_payload={"url": url, "parsed_video": resolved, "preloaded_context": preloaded_context},
    response_contract=response_contract,
    allowed_tools=["retrieval", "web_search"],
    required_tools=["retrieval"],
    strict_required_tool_order=True,  # 必须先 retrieval 再 web_search
    action_validator=video_analyze_action_validator,
)
```

`run_structured()` 的内部执行流程（见 `agents/llm_workspace_agent.py` 第 409 行）：

```
第1步：LLM 看当前状态，决定调用哪个工具
  ↓
第2步：如果 action == "final"，验证并返回结果
  ↓
第3步：如果是工具调用，检查：
  - 工具是否在 allowed_tools 中
  - 是否满足 required_tools 顺序
  - 是否超出工具预算（budget）
  - action_validator 是否通过
  ↓
第4步：调用工具（retrieval 或 web_search）
  ↓
第5步：工具返回结果（称为 observation）
  ↓
第6步：把 action 和 observation 加入 scratchpad（记忆）
  ↓
第7步：回到第1步，LLM 继续思考
  ↓
循环直到 LLM 返回 final
```

**⑥ Action 验证器**（`video_analyze_action_validator`）：
这是一个安全检查函数，确保 Agent：
- **禁止**调用 `hot_board_snapshot`（已在外部预加载）
- **禁止**调用 `video_briefing`（视频已解析）
- **必须**先调用 `retrieval` 拿到样本，才能调用 `web_search`
- `retrieval` 返回 >= 3 个样本时，**禁止**调用 `web_search`（节省 token）

class RetrievalTool(AgentTool):
    """从本地知识库检索案例、经验和结构化资料的工具。"""
    def __init__(self, description: str | None = None, timeout_seconds: float | None = None) -> None:
        super().__init__(

            description="从bilibili_knowledge中检索同垂类静态爆款样本",

`handler` 是工具的实际执行函数。`retrieval` 工具的 handler 调用 `knowledge_base.py` 中的 `kb_retrieve()` 函数，从 Chroma 向量数据库中检索与查询词最相似的文档。

**ReAct 循环**（第 148-207 行，`_run_once`）：
```python
def _run_once(self, step: int, budget: dict, ...) -> dict:
    # 构建 Prompt
    user_prompt = self._build_user_prompt(
        task_name, task_goal, user_payload,
        response_contract, required_tools,
        used_tools, scratchpad, history_block,
    )

    # LLM 决策
    decision = self.llm.invoke_json_required(system_prompt, user_prompt)
    action = decision["action"]
    action_input = decision["action_input"]

    # 如果是 final，验证并返回
    if action == "final":
        return self._validate_and_reflect(decision, ...)

    # 验证 action
    self._validate_action(action, budget, required_tools, ...)

    # 调用工具
    observation = self._invoke_tool_with_timeout(tool, action_input)

    # 记录到 scratchpad
    scratchpad.append({
        "action": action,
        "action_input": action_input,
        "observation": observation,
    })
```

**Reflection 自检**（第 216-240 行，`_reflect_final`）：
当 LLM 返回 final 后，Agent 会再调用一次 LLM 作为"质检员"，判断结果是否满足任务要求：
```python
review_prompt = f"""
你是结果质检器，判断下面这个JSON最终结果是否满足任务要求。
任务名称：{task_name}
响应契约：{response_contract}
工具观察：{scratchpad}
候选结果：{final}

如果满足，pass返回true。
如果不满足，pass返回false，并给出rewritten_final修正结果。
只返回JSON：{{"pass": boolean, "issues": [...], "rewritten_final": object|null}}
"""
```
这个设计确保 Agent 输出质量——类似于"让 LLM 检查 LLM 的输出"。

### 3.4 聊天 Agent 的工具

**文件**：`web/services/llm.py` 第 414-437 行

`get_llm_workspace_chat_agent()` 创建的 Agent 支持 4 个工具：

| 工具 | 何时使用 | Handler |
|------|---------|---------|
| `video_briefing` | 用户发送了 B站视频链接 | `video_briefing_tool_handler()` 解析视频信息 |
| `hot_board_snapshot` | 用户想看当前热点 | `hot_board_snapshot_tool_handler()` 获取热门榜 |
| `retrieval` | 用户要求回顾历史或检索知识库 | 从 Chroma 检索 |
| `web_search` | 用户想找视频或搜索内容 | 调用搜索工具 |

**系统提示词**（`build_workspace_chat_system_prompt`）的核心规则：
```
1. 回答时优先使用当前请求里已经提供的 creator_context、会话级历史和 video_url
2. 不要为了补全上下文默认调用 retrieval
3. 只有当用户明确要求回顾历史、检索知识库时，才允许调用 retrieval
4. 如果用户请求找视频、看内容、搜索推荐，优先调用 web_search
5. 如果工具超时、失败或无结果，直接跳过，基于现有上下文继续回答
```

### 3.5 会话历史存储

**文件**：`web/services/session_memory.py`

这是最容易困惑的部分，实际上是**三级缓存**：

```
第1级：Redis（内存数据库，速度最快）
    ↓ 未命中
第2级：进程内存（OrderedDict LRU缓存）
    ↓ 未命中
第3级：前端传入的历史（data["history"]）
    ↓ 还为空
第4级：空列表
```

**加载流程**：
```python
def load_session_history(self, session_id: str, frontend_history: object = None):
    key = f"session:{session_id}"

    # 优先查 Redis
    cached_payload, source = self._read_json(key, track_redis_miss=True)
    if cached_payload:
        history = normalize_chat_history(cached_payload.get("history"))
        return {"history": history, "source": "redis", ...}

    # Redis 未命中，查进程内存
    payload = self._memory_get(key)
    if payload:
        return {"history": payload, "source": "memory", ...}

    # 都未命中，用前端历史
    history = normalize_chat_history(frontend_history)
    return {"history": history, "source": "frontend_session", ...}
```

**会话持久化**（跨刷新不丢失）：
```python
# 异步保存到本地文件（chat_sessions/ 目录）
meta_store = get_chat_session_metadata_store()
meta_store.save_session_async(session_id, first_question, history)

# 文件结构：
# vector_db/chat_sessions/
#   sessions_index.json          ← 所有会话索引
#   {session_id}_meta.json       ← 会话元数据（创建时间、最后活跃时间等）
#   {session_id}_history.json    ← 会话历史
```
