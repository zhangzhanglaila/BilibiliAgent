# 前端说明

## 1. 当前前端结构

当前页面只保留两个模块，不再直接展示旧的多 Agent 工作台操作按钮。

页面文件：

- `web/templates/index.html`
- `web/static/app.js`
- `web/static/style.css`

## 2. 模块一：选题与文案

### 输入项

- `creatorField`：用户领域
- `creatorDirection`：内容方向
- `creatorIdea`：已有想法
- `creatorPartition`：可选分区，已扩展到知识、科技、生活、游戏、娱乐、美妆颜值、舞蹈、音乐、影视、动漫、美食、Vlog、穿搭、情感、萌宠、运动、汽车、商业职场
- `creatorStyle`：文案风格

### 前端行为

点击“一键生成选题与文案”后：

1. 前端收集输入
2. 调用 `/api/module-create`
3. 后端返回：
   - `normalized_profile`
   - `seed_topic`
   - `topic_result`
   - `copy_result`
   - `chosen_topic`
4. 前端渲染选题卡片和文案结果

## 3. 模块二：视频解析与优化

### 输入项

- `videoLink`：B 站视频链接

### 前端行为

点击“一键解析并分析视频”后：

1. 前端调用 `/api/module-analyze`
2. 后端解析链接并拿到视频数据
3. 返回：
   - `resolved`
   - `performance`
   - `topic_result`
   - `optimize_result`
   - `copy_result`（仅低表现视频时可能返回）
   - `analysis`
4. 前端根据 `performance.is_hot` 渲染不同结果区

## 4. 结果展示规则

### 模块一结果

包含：

- 整理后的方向
- 当前问题摘要
- 推荐主选题
- 热门选题建议
- 自动生成的标题、脚本、简介、标签、置顶评论

### 模块二结果

包含：

- 视频标题、UP 主、分区、BV 号
- 播放、点赞、投币、收藏、评论、分享
- 热门爆款 / 播放偏低 判断
- 爆款拆解或优化建议
- 后续可做方向
- 对低表现视频给出一版新的文案参考

## 5. 仍保留的兼容接口

虽然页面已经切到两个业务模块，但这些接口仍保留：

- `/api/topic`
- `/api/copy`
- `/api/operate`
- `/api/optimize`
- `/api/pipeline`

这样做是为了不破坏原有项目结构，也方便后续做二次开发。
