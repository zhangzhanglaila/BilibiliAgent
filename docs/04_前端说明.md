# 前端说明

## 1. 当前页面结构

当前前端不是旧版双面板悬停布局，而是稳定的三模块工作台。

页面分成 4 个区域：

1. 顶部状态区
2. 左侧模块切换导航
3. 左侧主工作区
4. 右侧智能对话助手

对应文件：

- [web/templates/index.html](../web/templates/index.html)
- [web/static/app.js](../web/static/app.js)
- [web/static/style.css](../web/static/style.css)

## 2. 顶部状态区

主要元素：

- `#globalStatusPill`
- `#statusText`
- `#clearResultsBtn`
- `#runtimeModePanel`
- `#runtimeModeToggle`
- `#runtimeConfigForm`

职责：

- 展示当前全局状态
- 展示运行模式
- 切换 LLM Agent 开关
- 保存运行时 LLM 配置
- 清空结果区

运行模式状态不是前端硬编码，而是服务端首屏下发：

- `window.__INITIAL_RUNTIME__`

同时也支持通过接口刷新：

- `GET /api/runtime-info`

## 3. 左侧模块区

当前通过顶部标签切换 3 个模块：

- `视频分析`
- `内容创作`
- `知识库管理`

切换逻辑在：

- `setActiveModule()`

当前模块容器：

- `#moduleGrid`

## 4. 视频分析模块

主要 DOM：

- `#videoLink`
- `#videoPreview`
- `#videoAnalyzeBtn`
- `#videoResult`

核心流程：

1. 用户输入视频链接
2. `scheduleVideoResolve()` 延迟触发解析
3. `resolveVideoLink()` 调 `/api/resolve-bili-link`
4. 结果先显示在 `#videoPreview`
5. 用户点击 `#videoAnalyzeBtn`
6. `runAnalyzeModule()` 调 `/api/module-analyze`
7. 渲染分析结果

当前重要特性：

- 支持自动解析，不必等到点分析才开始
- 支持标准链接、短链、`av` 链接、部分追踪链接
- 低表现视频会看到额外文案结果
- 参考视频卡片会显示更多指标

## 5. 内容创作模块

主要 DOM：

- `#creatorField`
- `#creatorDirection`
- `#creatorIdea`
- `#creatorPartition`
- `#creatorStyle`
- `#creatorRunBtn`
- `#creatorResult`

核心流程：

1. 填写创作输入
2. 点击一键生成
3. `runCreatorModule()` 调 `/api/module-create`
4. 渲染选题和文案

当前分区选择不是只有 5 个粗分区，还包含别名入口，例如：

- `beauty`
- `dance`
- `music`
- `film`
- `anime`
- `food`
- `vlog`
- `fashion`
- `emotion`
- `pet`
- `sports`
- `auto`
- `business`

这些会在后端归一到主分区。

## 6. 知识库管理模块

当前知识库模块已经是正式工作台的一部分，不是临时工具页。

### 6.1 子 Tab

当前有 4 个子功能：

- 上传知识文件
- 同步热门样本
- 查看知识库
- 关键词检索

主要容器：

- `#knowledgeStatusBar`
- `#knowledgeActionHost`
- `#knowledgeStageResult`

子 Tab 切换逻辑：

- `setKnowledgeSubtab()`

### 6.2 上传知识文件

接口：

- `POST /api/knowledge/upload`

支持格式：

- `.txt`
- `.md`
- `.docx`
- `.pdf`

### 6.3 同步热门样本

接口：

- `POST /api/knowledge/update`

当前同步源包括：

- 全站热门榜
- 每周必看
- 入站必刷
- 真实一级分区热门榜

### 6.4 查看知识库

接口：

- `GET /api/knowledge/sample`

### 6.5 关键词检索

接口：

- `GET /api/knowledge/search`

当前前端下拉框只展示一级分类，不再展示细分赛道。

当前分类包括：

- 番剧
- 国创
- 纪录片
- 电影
- 电视剧
- 综艺
- 动画
- 游戏
- 鬼畜
- 音乐
- 舞蹈
- 科技数码
- 汽车
- 时尚美妆
- 体育运动
- 动物
- 生活
- 知识科普
- 娱乐热点
- 职场成长
- 情感婚恋
- 两性心理
- 通用爆款

当前结果渲染还做了两件事：

- 按文档去重，避免同文档多个分片重复显示
- 标题优先显示“榜单名 / 分区 / 视频标题”，不再把内部 `BV...` 文档 ID 当标题

## 7. 智能助手

主要 DOM：

- `#assistantPanel`
- `#assistantResult`
- `#assistantMessage`
- `#assistantSendBtn`
- `#assistantLockOverlay`

接口：

- `POST /api/chat`

当前行为：

- 仅在 `LLM Agent 模式` 下可用
- 会把页面上下文一起传给后端
- 支持快捷提问按钮
- 支持浏览器语音输入

## 8. 当前前端状态管理重点

`web/static/app.js` 当前关键状态包括：

- `state.activeModule`
- `state.runtime`
- `state.videoResolved`
- `state.knowledgeActiveSubtab`
- `state.knowledgeForm`
- `state.knowledgeResults`
- `state.chatHistory`

## 9. 前端维护注意事项

### 9.1 三模块是当前真实结构

不要再按旧文档只写“两模块 + 聊天”。

### 9.2 知识库展示不能直接把分片当文档

否则会把同一篇文档的多个切片渲染成多条结果。

### 9.3 分类检索不是纯语义检索

后端会先扩召回，再做分类过滤，因此前端应继续把下拉值作为一级分类，不要再切回细分赛道直传。
