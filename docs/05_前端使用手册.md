# 前端使用手册

## 1. 如何打开

启动服务：

```bash
python web/app.py
```

浏览器访问：

```text
http://127.0.0.1:8000
```

打开页面后，先看顶部模式提示：

- `无 Key 规则模式`
- `LLM Agent 模式`

如果你想了解怎么切模式，直接看：

- [07_运行模式切换说明.md](D:/agent/docs/07_运行模式切换说明.md)

## 2. 模块一怎么用

适合这两类情况：

- 还没决定做什么内容
- 已经有模糊想法，但不知道怎么整理成容易起量的题

操作步骤：

1. 在“你的领域”里输入长期方向
2. 在“你想做的方向”里输入当前要做的内容类型
3. 在“你已有的想法”里输入一个问题、灵感或草稿
4. 选择内容分区和文案风格
5. 点击“一键生成选题与文案”

你会得到：

- 推荐主选题
- 3 个热门切口
- 标题备选
- 分段脚本
- 简介
- 标签
- 置顶评论

### 模块一在两种模式下的区别

#### 逻辑链路

- 结果来自本地规则引擎和模板
- 不消耗 token

#### LLM 链路

- 结果由 LLM Agent 根据工具返回数据实时生成
- 会消耗 token

## 3. 模块二怎么用

适合这几类情况：

- 视频已经发出去了
- 想知道这条视频更像爆款还是播放偏低
- 想知道下一步怎么改标题、封面、文案或选题

操作步骤：

1. 把 B 站视频链接粘贴到输入框
2. 等页面先自动解析当前视频信息
3. 确认预览区已经出现标题、分区、播放等信息
4. 点击“一键解析并分析视频”

### 重点变化

模块二现在是“先自动解析，再分析”，不是点按钮后才开始解析。

也就是说：

- 链接一贴上去就会尝试自动解析
- 没分析前也能先看到空框或真实解析值
- 鼠标放到字段卡片上会提示“根据视频链接自动解析”

### 自动解析前会看到什么

即使还没点分析按钮，预览区也会先展示这些字段的空框：

- 视频标题
- 视频类型 / 分区
- UP 主
- BV 号
- 播放量
- 点赞量
- 投币 / 收藏
- 评论 / 分享

### 分析后会看到什么

分析结果区会展示：

- 当前视频基础信息
- 热门 / 待优化判断
- 原因分析
- 标题与封面建议
- 内容节奏建议
- 后续可做选题
- 同类参考视频链接

### 模块二在两种模式下的区别

#### 逻辑链路

- 视频判断来自本地规则逻辑
- 优化建议来自纯代码链路
- 不消耗 token

#### LLM 链路

- 后端先解析真实视频数据
- 再由 LLM Agent 结合市场样本做判断和建议
- 会消耗 token

## 4. 智能对话助手怎么用

适合这几类情况：

- 不想自己判断该用模块一还是模块二
- 想直接问“帮我分析这个视频”
- 想让系统自己决定先查什么数据、再给什么结论

操作步骤：

1. 先确认顶部显示的是 `LLM Agent 模式`
2. 在右侧聊天输入框输入自然语言问题
3. 点击“发送给智能助手”

当前聊天区是标准会话形式：

- 用户消息显示在右边
- 助手回复显示在左边
- 输入框固定在底部
- 历史消息会持续保留在对话区

助手回复里可能包含：

- 直接答案
- 下一步建议
- 工具调用痕迹
- 可点击跳转的参考视频链接

## 5. 页面交互说明

### 桌面端模块伸缩

桌面端默认：

- 模块一更宽
- 模块二更窄

当鼠标移到某个模块时：

- 当前模块自动变宽
- 另一个模块自动收窄

这样做是为了让你在当前工作区有更大的可视面积。

### 进度条

模块一、模块二、聊天助手执行时都会显示进度条：

- 显示处理中百分比
- 显示当前阶段文字
- 避免用户误以为卡死

## 6. 常见问题

### 6.1 没填很多内容也能生成吗

可以。

模块一里，领域、方向、想法三项至少填一项就可以开始。

### 6.2 支持什么链接格式

支持：

- 标准 B 站视频链接
- 常见 `b23.tv` 短链

### 6.3 如果自动解析失败怎么办

优先检查：

1. 链接能不能在浏览器正常打开
2. 当前网络能不能访问 B 站
3. 该视频是不是公开视频

### 6.4 为什么聊天面板是灰的

因为当前运行在逻辑链路。

只有配置 `LLM_API_KEY` 后，聊天助手才会启用。

### 6.5 什么时候会消耗 token

只有页面处于 `LLM Agent 模式` 时才会消耗 token。

也就是：

- 模块一生成
- 模块二分析
- 聊天助手对话

这三个入口都会调用模型。
