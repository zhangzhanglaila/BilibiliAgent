# 前端使用手册

这份手册专门讲：页面怎么点、每个按钮是什么意思、结果怎么看。

---

## 1. 打开页面

先在项目根目录执行：

```bash
python web/app.py
```

然后浏览器打开：

```text
http://127.0.0.1:8000
```

你会看到一个单页操作台。

---

## 2. 页面区域说明

页面主要分两部分：

### 上半部分：输入与操作区
这里负责填写参数、点击按钮。

### 下半部分：结果区
这里负责显示运行状态和输出结果。

---

## 3. 输入框说明

### 3.1 分区
输入框名称：`分区`

作用：
- 告诉选题 Agent 你想看哪个分区的内容热点

默认值：
```text
knowledge
```

常见可填：
- `knowledge`
- `tech`
- `life`
- `game`
- `ent`

如果你不确定，就先用 `knowledge`。

---

### 3.2 同类 UP 主 ID
输入框名称：`同类 UP 主 ID`

作用：
- 让选题 Agent 参考同类账号的爆款视频

格式：
- 多个 ID 用英文逗号分隔

示例：
```text
546195,15263701,777536
```

如果你暂时没有自己的对标账号，可以直接用默认值。

---

### 3.3 文案风格
输入框名称：`文案风格`

可选项：
- 干货
- 教学
- 搞笑
- 混剪

作用：
- 控制文案 Agent 输出的语言风格

建议：
- 新手先选“干货”
- 做教程类内容选“教学”
- 做轻松内容选“搞笑”
- 做片段拼接类选“混剪”

---

### 3.4 视频 BV 号
输入框名称：`视频 BV 号`

作用：
- 给运营 Agent 和优化 Agent 指定目标视频

示例：
```text
BV1xx411c7mD
```

如果输入无效，程序会尽量使用 fallback 数据继续跑流程。

---

### 3.5 手动选题
输入框名称：`手动选题`

作用：
- 给文案 Agent 直接传主题

示例：
```text
AI 视频剪辑提效
```

当你不想先跑选题 Agent 时，可以直接填这里。

---

## 4. 按钮说明

### 4.1 运行选题 Agent
按钮名称：`运行选题 Agent`

作用：
- 分析全站热点 + 分区热点 + 同类 UP 主内容
- 输出 3 个高潜力选题

适合什么时候点：
- 想找下一个视频做什么
- 想看近期热点方向

结果里你会看到：
- topic
- reason
- video_type
- keywords

---

### 4.2 运行文案 Agent
按钮名称：`运行文案 Agent`

作用：
- 根据手动选题生成文案

适合什么时候点：
- 你已经知道要做什么主题
- 你只想快速生成标题和脚本

结果里你会看到：
- titles
- script
- description
- tags
- pinned_comment

---

### 4.3 运行运营 Agent
按钮名称：`运行运营 Agent`

作用：
- 分析指定 BV 视频评论
- 生成回复建议
- 识别垃圾评论
- 给出点赞、关注建议

注意：
- 默认是 dry-run
- 页面展示的是“建议动作”，不是直接帮你执行账号操作

结果里你会看到：
- replies
- deletions
- likes
- follows
- summary

---

### 4.4 运行优化 Agent
按钮名称：`运行优化 Agent`

作用：
- 拉取视频数据
- 分析为什么流量不足
- 输出优化建议

结果里你会看到：
- diagnosis
- optimized_titles
- cover_suggestion
- content_suggestions
- benchmark_summary

---

### 4.5 一键运行全流程
按钮名称：`一键运行全流程`

作用：
- 一次串联 4 大 Agent
- 直接得到完整结果

流程是：

```text
选题 -> 文案 -> 运营 -> 优化
```

适合什么时候点：
- 演示项目
- 快速看全链路效果
- 验证整个系统是否正常

---

## 5. 结果查看方式

页面下方有一个结果区。

### 5.1 状态显示
你会看到类似：
- 等待执行
- 正在执行 topic...
- topic 执行完成
- pipeline 执行失败

这表示当前请求状态。

### 5.2 结果展示
当前结果区显示的是 JSON 文本。

优点：
- 对开发者友好
- 结构清晰
- 方便后续复制和调试

你可以直接看字段值。

例如：
- `ideas`：选题列表
- `titles`：标题列表
- `summary`：运营汇总
- `diagnosis`：优化诊断

---

## 6. 推荐操作顺序

如果你是第一次使用，建议这样点：

### 顺序 1：只体验选题
1. 填分区
2. 点击“运行选题 Agent”
3. 看是否能拿到选题建议

### 顺序 2：体验文案
1. 在“手动选题”填一个主题
2. 选择风格
3. 点击“运行文案 Agent”

### 顺序 3：体验数据优化
1. 输入 BV 号
2. 点击“运行优化 Agent”

### 顺序 4：最后跑全流程
1. 填好所有参数
2. 点击“一键运行全流程”

这样最容易理解。

---

## 7. 常见问题

### 问题 1：我点了按钮没有结果
先检查：
- 后端服务有没有启动
- 页面是否是 `http://127.0.0.1:8000`
- 控制台是否报错

### 问题 2：结果是 JSON，不够好看怎么办
这是当前版本的设计选择。

目的是：
- 先把功能跑通
- 便于开发者理解 API 返回内容

后续你可以自己改成卡片、表格、标签页。

### 问题 3：为什么互动不是自动真实执行？
因为默认是安全模式。

当前页面主要用于：
- 生成建议动作
- 验证逻辑
- 避免误操作账号

### 问题 4：没有填 LLM Key 还能不能用？
可以。

系统会走 fallback 模板逻辑。

---

## 8. 给零基础用户的建议

如果你第一次用，不要一下子理解所有代码。

先记住这三件事：
1. 上面填参数
2. 中间点按钮
3. 下面看结果

你能先把它用起来，再慢慢研究代码。

---

## 9. 一句话总结

这个页面的目标很明确：
**把原来只能用命令行操作的多 Agent 系统，变成“看得见、点得动、能直接演示”的浏览器页面。**

对 Java 开发者来说，这会大幅降低你理解整个项目的门槛。
