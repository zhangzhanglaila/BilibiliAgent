# 项目说明

## 1. 项目定位

这个项目面向 B 站创作者的两个高频场景：

1. 还没发布视频，不知道做什么内容
2. 已经发布了视频，想分析和优化

项目不是一个泛泛的 AI Demo，而是把创作前和发布后的两段工作流拆开，做成两个可以直接使用的模块。

## 2. 当前产品结构

### 模块一：选题与文案生成

适用人群：

- 还没有明确选题
- 只有模糊方向和想法
- 想快速得到一套能直接使用的文案

输入：

- 领域
- 方向
- 想法
- 可选分区
- 文案风格

输出：

- 3 个高潜力选题
- 自动生成的标题
- 视频脚本
- 简介、标签、置顶评论

### 模块二：视频解析与优化

适用人群：

- 已经发了视频
- 想判断视频更像爆款还是低表现内容
- 想得到下一步具体优化建议

输入：

- B 站视频链接

输出：

- 视频基础信息解析
- 播放、点赞、投币、收藏等数据
- 热门爆款 / 播放偏低 判断
- 爆款原因拆解或低表现优化建议

## 3. 保留的底层 Agent

虽然前端产品已经改成两个模块，但底层仍保留 4 个 Agent：

### 3.1 TopicAgent

文件：`agents/topic_agent.py`

职责：

- 抓取热榜、分区热点、同类 UP 样本
- 围绕当前输入主题生成更值得做的选题

### 3.2 CopywritingAgent

文件：`agents/copywriting_agent.py`

职责：

- 根据选题生成标题、脚本、简介、标签、置顶评论

### 3.3 OperationAgent

文件：`agents/operation_agent.py`

职责：

- 分析评论
- 给出回复、点赞、关注、过滤建议

当前 Web 端主流程没有把它单独暴露成独立模块，但底层接口仍保留。

### 3.4 OptimizationAgent

文件：`agents/optimization_agent.py`

职责：

- 拉取目标视频数据
- 结合样本给出标题、封面、内容优化建议

## 4. 架构关系

```text
前端页面
  -> web/app.py
  -> main.py
  -> graph.py
  -> TopicAgent / CopywritingAgent / OperationAgent / OptimizationAgent
  -> B站公开数据 / SQLite / LLM
```

## 5. 当前边界

- 项目强调“可运行、可学习、可扩展”
- 并不是完整商用 SaaS 后台
- 某些 B 站公开接口可能失效，因此 Web 层有多种解析回退
- 如果没有 LLM Key，会自动走本地降级逻辑

## 6. 适合谁

适合：

- B 站内容创作者
- 想做 Agent 项目的开发者
- 想从 Java / SpringBoot 过渡到 Python Agent 项目的开发者

不太适合：

- 需要多租户权限体系的大型后台系统
- 需要强一致高并发调度的生产平台
