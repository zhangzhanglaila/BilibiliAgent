# 项目说明文档

## 1. 项目定位
这是一个面向 B 站 UP 主的全自动运营多 Agent 系统。

它解决的不是“怎么写一段 AI demo”，而是“怎么把选题、写文案、互动运营、数据复盘串起来”。

适合拿来做：
- 个人 UP 主运营助手
- AI + 内容创作练手项目
- LangGraph 多 Agent 入门项目
- Java 开发者转 Python / AI Agent 的实战项目

---

## 2. 核心功能
项目围绕 4 个核心 Agent 展开：

1. **选题 Agent**
   - 抓取 B 站公开热榜、指定分区热点、同类 UP 主视频
   - 分析播放、点赞、收藏、评论等指标
   - 输出 3 个高潜力选题

2. **文案 Agent**
   - 根据选题自动生成标题、脚本、简介、标签、置顶评论
   - 支持干货、教学、搞笑、混剪等风格
   - 支持 LLM 与本地降级模板双模式

3. **运营 Agent**
   - 自动分析评论内容
   - 生成回复建议
   - 过滤垃圾评论
   - 默认 dry-run，避免误操作

4. **数据优化 Agent**
   - 拉取视频核心数据并存入 SQLite
   - 对比同类样本做诊断
   - 输出标题、封面、内容优化建议

---

## 3. 四大 Agent 介绍

### 3.1 选题 Agent
对应文件：`agents/topic_agent.py`

职责：
- 找热点
- 找同类爆款
- 找值得做的新选题

输出示例：
- 选题名称
- 推荐理由
- 适合的视频类型
- 关键词

### 3.2 文案 Agent
对应文件：`agents/copywriting_agent.py`

职责：
- 根据选题生成完整发布文案
- 支持直接输入主题
- 支持多种视频风格

输出内容：
- 3 个标题
- 分段脚本
- 视频简介
- 标签
- 置顶评论

### 3.3 运营 Agent
对应文件：`agents/operation_agent.py`

职责：
- 读取视频评论
- 判断是否垃圾评论
- 生成回复建议
- 给出点赞、关注建议动作

注意：
- 默认是安全模式，不直接执行敏感操作
- 更适合做“运营辅助决策”而不是直接代替人工账号管理

### 3.4 数据优化 Agent
对应文件：`agents/optimization_agent.py`

职责：
- 拉取指定 BV 视频数据
- 写入 SQLite 历史表
- 对比历史与样本
- 输出优化建议

输出内容：
- 数据诊断
- 2 个优化标题
- 封面方向建议
- 内容调整建议

## 5. 项目亮点

### 亮点 1：能跑通
不是只讲概念，项目有完整入口、完整 Agent、完整前端页面。

### 亮点 2：依赖少
没有引入 Redis、MySQL、Celery、Vue、React、Node 打包链。

### 亮点 3：新手友好
即使没有 LLM Key，也能通过降级逻辑完整跑流程。

### 亮点 4：适合 Java 开发者过渡
整体结构接近“分层 + 编排 + API + 页面”的后端工程思路。

### 亮点 5：可二次开发
每个 Agent 单独封装，后续可以继续接入：
- 更多平台
- 更多模型
- 真正的登录态操作
- 定时调度

## 6. 适用人群

适合：
- B 站内容创作者
- 想做 AI Agent 项目的开发者

## 7. 项目架构图（文字版）

```text
浏览器前端页面
   ↓
Flask Web 层（web/app.py）
   ↓
可复用运行函数（main.py）
   ↓
LangGraph 编排层（graph.py）
   ↓
4 大 Agent
   ├─ TopicAgent：选题分析
   ├─ CopywritingAgent：文案生成
   ├─ OperationAgent：互动运营
   └─ OptimizationAgent：数据优化
   ↓
基础能力层
   ├─ llm_client.py：模型调用
   ├─ db.py：SQLite 存储
   ├─ config.py：配置管理
   └─ models.py：数据结构
   ↓
B 站公开数据 / 本地 SQLite / 大模型接口
```

## 10. 项目当前边界说明

这个项目强调“可跑通、可学习、可扩展”，不是直接商用版系统。

当前边界：
- 完播率是估算值
- 某些 B 站高权限操作默认 dry-run
- LLM 如果没有 Key，会走模板降级
- 前端是轻量页面，不是复杂后台管理系统

但这正适合学习和二次开发。
