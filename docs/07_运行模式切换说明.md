# 运行模式切换说明

这份文档面向普通用户，专门说明：

- 怎么切到逻辑链路（无 Key）
- 怎么切到 LLM 链路（有 Key）
- 切换之后去哪里确认当前实际生效的是哪条链路

## 1. 两种模式先看懂

### 逻辑链路（无 Key）

特点：

- 不需要配置 `LLM_API_KEY`
- 模块一和模块二走现有纯代码规则
- 聊天助手关闭
- 不消耗 token

### LLM 链路（有 Key）

特点：

- 需要配置 `LLM_API_KEY`
- 模块一和模块二切到 `LLMWorkspaceAgent`
- 聊天助手开启
- 会消耗 token

## 2. 切到逻辑链路（无 Key）怎么做

最简单的做法就是让 `.env` 里的 `LLM_API_KEY` 为空。

### 方法一：直接清空 Key

打开：

- [`.env`](D:/agent/.env)

把这一行改成：

```env
LLM_API_KEY=
```

然后重启服务。

### 方法二：临时删除 `.env`

如果你不想保留本地配置，也可以直接删除：

- [`.env`](D:/agent/.env)

然后重启服务。

### 切换完成后会发生什么

- 模块一走逻辑链路
- 模块二走逻辑链路
- 聊天助手会变成不可用状态
- 不会消耗 token

## 3. 切到 LLM 链路（有 Key）怎么做

打开：

- [`.env`](D:/agent/.env)

至少确保下面 3 项有值：

```env
LLM_API_KEY=你的真实 Key
LLM_BASE_URL=你的模型服务地址
LLM_MODEL=你的模型名
```

例如：

```env
LLM_PROVIDER=openai
LLM_API_KEY=你的真实 Key
LLM_BASE_URL=https://your-base-url/v1
LLM_MODEL=你的模型名
```

然后重启服务。

### 切换完成后会发生什么

- 模块一切到 LLM 链路
- 模块二切到 LLM 链路
- 聊天助手启用
- 会开始消耗 token

## 4. 改完 `.env` 之后为什么还没切换

最常见原因只有一个：

- 你改了 `.env`，但没有重启 Python 服务

这个项目在启动时读取环境变量，所以切换模式后一定要重启：

```powershell
D:\agent\.venv\Scripts\python.exe D:\agent\web\app.py
```

如果你平时用的是：

```powershell
python web\app.py
```

那也一样，改完 `.env` 后要重新启动。

## 5. 怎么确认当前实际走的是哪条链路

### 方法一：看页面顶部

页面顶部会直接显示：

- `运行模式：无 Key 规则模式`
- 或 `运行模式：LLM Agent 模式`

这是普通用户最直观的确认方式。

### 方法二：看聊天面板

- 聊天面板灰掉、不可发送 -> 逻辑链路
- 聊天面板可输入、可发送 -> LLM 链路

### 方法三：看接口返回

打开：

```text
http://127.0.0.1:8000/api/runtime-info
```

如果返回：

- `mode: "rules"` -> 逻辑链路
- `mode: "llm_agent"` -> LLM 链路

## 6. 一般用户推荐怎么切

### 你只是想本地先跑通项目

建议：

- 用逻辑链路
- 不填 `LLM_API_KEY`

原因：

- 稳定
- 没有额外成本
- 先把功能和页面跑通最容易

### 你想要更强的分析、生成和聊天能力

建议：

- 用 LLM 链路
- 填好 `LLM_API_KEY`
- 再重启服务

原因：

- 模块一和模块二都会切到 LLM Agent
- 聊天助手也会启用

## 7. 一个最短操作清单

### 切到逻辑链路

1. 打开 [`.env`](D:/agent/.env)
2. 把 `LLM_API_KEY` 清空
3. 重启服务
4. 看页面顶部是否显示 `无 Key 规则模式`

### 切到 LLM 链路

1. 打开 [`.env`](D:/agent/.env)
2. 填写 `LLM_API_KEY`、`LLM_BASE_URL`、`LLM_MODEL`
3. 重启服务
4. 看页面顶部是否显示 `LLM Agent 模式`

## 8. 最后一句

切换模式的唯一核心开关就是：

`LLM_API_KEY`

有 Key，就走 LLM 链路。

没 Key，就走逻辑链路。
