# 运行模式切换说明

这份文档只回答一件事：

- 当前项目怎么在规则模式和 LLM Agent 模式之间切换

并且告诉你：

- 切完以后去哪里确认真的生效了

## 1. 两种模式分别是什么

### 1.1 规则模式

触发条件：

- `.env` 中没有配置 `LLM_API_KEY`

效果：

- 内容创作模块走规则链路
- 视频分析模块走规则链路
- 智能对话助手关闭
- 主界面不消耗 token

### 1.2 LLM Agent 模式

触发条件：

- `.env` 中配置了 `LLM_API_KEY`

效果：

- 内容创作模块切到 `LLMWorkspaceAgent`
- 视频分析模块切到 `LLMWorkspaceAgent`
- 智能对话助手启用
- 主界面开始消耗 token

## 2. 切到规则模式怎么做

最直接的方法就是清空 `.env` 里的 `LLM_API_KEY`。

打开：

- [`.env`](D:/agent/.env)

把它改成：

```env
LLM_API_KEY=
```

然后重启服务。

### 2.1 切到规则模式后会发生什么

- 顶部显示“无 Key 规则模式”
- 右侧助手不可用
- 左侧两个模块仍然可用
- 主界面不消耗 token

## 3. 切到 LLM Agent 模式怎么做

打开：

- [`.env`](D:/agent/.env)

至少确保下面这些字段有值：

```env
LLM_PROVIDER=openai
LLM_API_KEY=你的真实 Key
LLM_BASE_URL=https://zapi.aicc0.com/v1
LLM_MODEL=你的模型名
```

然后重启服务。

### 3.1 典型示例

```env
LLM_PROVIDER=openai
LLM_API_KEY=your-real-key
LLM_BASE_URL=https://zapi.aicc0.com/v1
LLM_MODEL=gpt-5.4
```

或者如果你的 provider 对更轻量模型更稳定，也可以改成：

```env
LLM_MODEL=gpt-4.1-mini
```

前提是该 provider 支持这个模型名。

### 3.2 切到 LLM 模式后会发生什么

- 顶部显示“LLM Agent 模式”
- 右侧助手启用
- 内容创作和视频分析切到 LLM Agent 链路
- 主界面开始消耗 token

## 4. 改了 `.env` 为什么没生效

最常见原因只有一个：

- 你改了 `.env`
- 但没有重启 Python 服务

这个项目在启动时读取环境变量，所以改完必须重启。

例如：

```powershell
python web\app.py
```

或者：

```powershell
D:\agent\.venv\Scripts\python.exe D:\agent\web\app.py
```

## 5. 怎么确认当前实际走的是哪种模式

最推荐用下面 3 种方法。

### 5.1 看页面顶部

页面顶部会直接显示：

- `无 Key 规则模式`
- 或 `LLM Agent 模式`

这是最直观的方法。

### 5.2 看右侧聊天助手

- 灰掉、不能发送 -> 规则模式
- 可输入、可发送 -> LLM Agent 模式

### 5.3 看运行时接口

打开：

```text
http://127.0.0.1:8000/api/runtime-info
```

如果返回：

- `mode: "rules"` -> 规则模式
- `mode: "llm_agent"` -> LLM Agent 模式

## 6. 如何判断当前切换成功但 provider 本身不可用

这是很重要的区分。

有时候你会遇到：

- 页面顶部已经是 `LLM Agent 模式`
- 但分析或聊天仍然失败

这说明：

- 模式切换已经成功
- 失败点在 LLM provider、模型名、网关或超时

不是“没有切到 LLM 模式”。

## 7. 如果切到 LLM 模式后变慢了怎么办

先检查这几项：

- `LLM_BASE_URL`
- `LLM_MODEL`
- `LLM_TIMEOUT_SECONDS`
- `LLM_MAX_RETRIES`

如果当前 provider 对复杂请求慢，可以尝试：

1. 换更快的模型
2. 调小重试次数
3. 视情况调大超时

## 8. 最短操作清单

### 8.1 切到规则模式

1. 打开 [`.env`](D:/agent/.env)
2. 清空 `LLM_API_KEY`
3. 重启服务
4. 看页面顶部是否显示规则模式

### 8.2 切到 LLM Agent 模式

1. 打开 [`.env`](D:/agent/.env)
2. 填写 `LLM_API_KEY`
3. 填写 `LLM_BASE_URL`
4. 填写 `LLM_MODEL`
5. 重启服务
6. 看页面顶部是否显示 `LLM Agent 模式`

## 9. 最后一句

切换主开关只有一个：

- `LLM_API_KEY`

但切到 LLM 模式后能不能稳定工作，还取决于：

- `LLM_BASE_URL`
- `LLM_MODEL`
- 当前 provider 的实际可用性
