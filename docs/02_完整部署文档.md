# 完整部署文档

## 1. 环境要求

- Python 3.10+
- Windows / macOS / Linux
- 可访问 B 站公开页面和公开接口

## 2. 安装依赖

```bash
pip install -r requirements.txt
```

## 3. 配置环境变量

先复制模板：

### Windows

```bash
copy .env.example .env
```

### macOS / Linux

```bash
cp .env.example .env
```

常见配置项：

- `LLM_API_KEY`
- `LLM_BASE_URL`
- `LLM_MODEL`
- `DEFAULT_PARTITION`
- `DEFAULT_PEER_UPS`
- `DB_PATH`

说明：

- 没有 `LLM_API_KEY` 也能运行
- 没有 Key 时，系统走逻辑链路（纯代码规则模式）
- 配置 Key 后，系统走 LLM 链路（LLM Agent 模式）

## 4. 启动 Web 服务

```bash
python web/app.py
```

浏览器访问：

```text
http://127.0.0.1:8000
```

## 5. 当前页面包含的功能区

### 模块一：还没发布视频

用于：

- 输入领域、方向、想法
- 生成选题和文案

### 模块二：已经发布视频

用于：

- 输入 B 站视频链接
- 自动解析视频信息
- 判断热门爆款还是播放偏低
- 返回拆解或优化建议

### 智能对话助手

用于：

- 直接自然语言提问
- 让 Agent 自主调用工具回答

注意：

- 只有在配置 `LLM_API_KEY` 后才会启用

## 6. 运行模式说明

### 逻辑链路（无 Key）

- 模块一：`/api/module-create` -> 纯代码选题与文案链路
- 模块二：`/api/module-analyze` -> 纯代码解析、判断、优化链路
- 聊天助手：关闭

### LLM 链路（有 Key）

- 模块一：`/api/module-create` -> `LLMWorkspaceAgent` + `creator_briefing` 工具
- 模块二：`/api/module-analyze` -> `LLMWorkspaceAgent` + `video_briefing` 工具
- 聊天助手：`/api/chat` -> `LLMWorkspaceAgent` 自主调用工具

如果你想看普通用户怎么手动切换模式，直接看：

- [07_运行模式切换说明.md](D:/agent/docs/07_运行模式切换说明.md)

## 7. CLI 调试命令

### 选题

```bash
python main.py topic --partition knowledge --topic "AI 剪辑效率"
```

### 文案

```bash
python main.py copy --topic "AI 剪辑第一条视频先拍什么更容易起量" --style 干货
```

### 视频优化

```bash
python main.py optimize --bv BV1xx411c7mD
```

### 全流程

```bash
python main.py pipeline --bv BV1xx411c7mD --partition knowledge --style 干货 --topic "AI 剪辑效率"
```

## 8. 常见问题

### 7.1 链接能解析 BV，但视频信息解析失败

说明：

- B 站公开接口可能对某些视频返回 `-404`
- 项目会自动尝试多种回退方式
- 如果仍失败，通常是当前网络环境无法正常访问对应页面

### 7.2 没有 LLM Key，结果会不会不可用

不会。

只是 Web 端会走逻辑链路，聊天助手不可用，但页面和主流程仍可直接使用。

### 7.3 为什么会生成本地数据库

默认会在项目根目录生成：

```text
bilibili_agents.db
```

用于保存部分视频数据和优化历史。
