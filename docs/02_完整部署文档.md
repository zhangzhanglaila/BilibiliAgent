# 完整部署文档

## 1. 环境要求

- Python 3.10 及以上
- Windows / macOS / Linux
- 能访问 B 站公开接口
- 如果启用 LLM，需要能访问你配置的模型服务地址

推荐在独立虚拟环境中安装。

## 2. 安装依赖

项目根目录执行：

```bash
pip install -r requirements.txt
```

当前依赖以 [requirements.txt](../requirements.txt) 为准，核心包括：

- `flask`
- `bilibili-api-python`
- `chromadb`
- `langchain-chroma`
- `langgraph`
- `sentence-transformers`
- `python-docx`
- `pypdf`

## 3. 初始化 `.env`

复制模板：

### Windows

```bash
copy .env.example .env
```

### macOS / Linux

```bash
cp .env.example .env
```

模板文件见：[.env.example](../.env.example)

## 5.1 下次自己替换 AI Key 的最短流程

只改项目根目录的 `.env`：

1. 替换 `LLM_API_KEY`
2. 同步替换 `OPENAI_API_KEY`
3. 如供应商或网关有变化，再替换 `LLM_PROVIDER`
4. 如接口地址有变化，再替换 `LLM_BASE_URL`
5. 如模型有变化，再替换 `LLM_MODEL`
6. 如要调低推理成本或速度，再改 `LLM_REASONING_EFFORT`
7. 重启 `python web/app.py`

补充说明：

- 页面运行时表单适合临时改 `provider / url / key / model`
- `.env` 适合保存长期配置
- `LLM_REASONING_EFFORT`、`LLM_DISABLE_RESPONSE_STORAGE` 当前以 `.env` 为准

## 5.2 LangSmith tracing

当前项目已接入 LangSmith，可选开启。

启用后会自动记录：

- 所有 `LLMClient` 发出的模型调用
- 知识库 `retrieve()` 检索
- `LLMWorkspaceAgent` 的多步工具/RAG 流程
- Web 的 `module-create / module-analyze / chat` 顶层链路

### 6.1 Web

```bash
python web/app.py
```


当前 CLI 子命令以 [main.py](../main.py) 为准：

## 7. 首次启动检查项

建议按下面顺序检查：

1. 页面是否能打开 `http://127.0.0.1:8000`
2. 顶部运行模式是否和预期一致
3. 视频链接解析是否正常
4. 知识库状态是否能正常读取
5. 如果需要 LLM，运行模式区域是否能成功保存配置并切换

## 8. 当前知识库部署注意事项

### 8.1 知识库数据目录

知识库默认写到：

```text
./vector_db
```

如果需要迁移目录，改 `VECTOR_DB_PATH` 即可。

当前项目知识库后端优先使用 Chroma。

如果运行环境里缺少 `chromadb` 或 `langchain-chroma`：

- 系统会自动回退到 `vector_db` 目录下的本地 JSON 存储
- 上传、同步、查看、检索仍可继续使用
- 但如果你希望恢复 Chroma，就需要把 `requirements.txt` 里的相关依赖装齐

### 8.2 Embedding 模型

项目优先使用 `sentence-transformers`。

如果本地模型不可用：

- 不会导致知识库彻底不可用
- 会自动回退到 deterministic embedding
- 只是召回效果会比正常 embedding 弱

### 8.3 文件导入格式

当前支持：

- `.txt`
- `.md`
- `.docx`
- `.pdf`

## 9. 生产部署建议

- 把 `.env` 放到服务器本地，不要进仓库
- `VECTOR_DB_PATH` 放到独立持久化目录
- `model_cache` 单独持久化，避免重复下载 embedding 模型
- 使用反向代理托管 Flask 服务
- 如果要启用 LLM，优先先在页面里验证运行时配置，再切正式流量
