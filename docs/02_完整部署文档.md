# 从零到一完整部署文档

这份文档适合：
- 第一次运行 Python 项目的人
- Java / SpringBoot 开发者
- 想在 Windows 或 macOS 上快速跑起本项目的人

---

## 1. 环境要求

推荐环境：
- Python 3.10 及以上
- pip 可用
- 能访问互联网下载依赖

项目依赖：
- LangGraph
- LangChain
- bilibili-api-python
- Flask
- SQLite（Python 自带，无需单独安装）

---

## 2. Windows 部署流程

### 2.1 安装 Python
1. 打开 Python 官网下载安装 Python 3.10+
2. 安装时勾选：
   - `Add Python to PATH`
3. 安装完成后打开终端执行：

```bash
python --version
```

如果能看到类似：

```bash
Python 3.10.0
```

说明安装成功。

### 2.2 进入项目目录

```bash
cd D:\agent
```

### 2.3 创建虚拟环境

```bash
python -m venv .venv
```

### 2.4 激活虚拟环境

```bash
.venv\Scripts\activate
```

激活成功后，命令行前面一般会出现 `(.venv)`。

### 2.5 安装依赖

```bash
pip install -r requirements.txt
```

### 2.6 配置环境变量

把 `.env.example` 复制为 `.env`：

```bash
copy .env.example .env
```

然后按需修改 `.env`：

```env
LLM_API_KEY=
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_MODEL=deepseek-chat
DB_PATH=bilibili_agents.db
REQUEST_INTERVAL=1.2
```

说明：
- 不填 `LLM_API_KEY` 也能运行
- 不填时文案与优化建议会走本地模板模式

## 3. macOS 部署流程

### 3.1 检查 Python

```

- 官网安装包

### 3.2 进入项目目录

### 3.3 创建虚拟环境

python3 -m venv .venv

pip install -r requirements.txt
```

### 3.6 配置环境变量

然后编辑 `.env` 文件。

## 4. 环境变量说明

### 必填吗？
不是。

### 常用项
- `LLM_API_KEY`：模型密钥
- `LLM_BASE_URL`：模型接口地址
- `LLM_MODEL`：模型名
- `DB_PATH`：SQLite 文件路径
- `REQUEST_INTERVAL`：请求间隔，避免限流
- `DEFAULT_PARTITION`：默认分区
- `DEFAULT_PEER_UPS`：默认同类 UP 主 ID

### 什么时候需要填 Key？
如果你想让：
- 文案更自然
- 标题更像真人写的
- 优化建议更像 AI 诊断

就建议填。

如果只是学习项目结构，不填也完全可以。

## 5. 项目启动方式

### 5.1 命令行启动

#### 运行选题 Agent
```bash
python main.py topic --partition knowledge
```

#### 运行文案 Agent
```bash
python main.py copy --topic "AI 视频剪辑提效" --style 干货
```

#### 运行运营 Agent
```bash
python main.py operate --bv BV1xx411c7mD --dry-run
```

#### 运行数据优化 Agent
```bash
python main.py optimize --bv BV1xx411c7mD
```

#### 一键运行全流程
```bash
python main.py pipeline --bv BV1xx411c7mD --partition knowledge --style 干货
```

---

### 5.2 前端启动

执行：

```bash
python web/app.py
```

看到类似输出：

```bash
 * Running on http://127.0.0.1:8000
```

然后浏览器打开：

```text
http://127.0.0.1:8000
```

就能看到前端操作页面。

## 8. 常见问题排查

### 问题 1：`python` 命令找不到
原因：
- Python 没装好
- PATH 没配置

解决：
- 重新安装 Python，并勾选 `Add Python to PATH`
- 或使用 `python3`

### 问题 2：依赖安装慢或失败
原因：
- 网络问题
- pip 源慢

解决：
- 重试安装
- 或改用镜像源

例如：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 问题 3：没有 LLM Key，文案是不是不能用？
不是。

项目已经内置 fallback 模式，没有 Key 也能输出模板化结果。

### 问题 4：B 站接口请求失败
可能原因：
- 网络波动
- B 站接口限制
- 公开接口数据结构变化

解决：
- 稍后重试
- 使用默认 fallback 演示数据继续学习
- 提高 `REQUEST_INTERVAL`

### 问题 5：为什么有些互动功能没有真的执行？
因为项目默认是 `dry-run`。

这是为了避免：
- 误删评论
- 误点赞
- 误关注

当前设计是“先给建议动作，再考虑接入真实登录态执行”。

### 问题 6：前端打不开
检查：
1. 是否执行了 `python web/app.py`
2. 是否访问了 `http://127.0.0.1:8000`
3. 8000 端口是否被占用

### 问题 7：页面有结果但看不懂
正常。

页面当前直接展示结构化 JSON，优点是：
- 开发阶段直观
- 便于二次开发
- 便于 Java 开发者理解后端返回格式
